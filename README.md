## Descrição

O desafio consiste em elaborar um sistema de Gestão de Cadastros contendo:

1. Dados de clientes, incluindo endereços.
2. Dados de produtos
3. Dados armazenados no local storaga.

Essa API foi criada usando HTML, JavaScript e [Bootstrap](https://getbootstrap.com/)

### Como Usar

Foi feita uma interface simples que permite executar operações basicas de manipulação de dados **(adicionar, atualizar, deletar e listar)**. 

Para cadastrar um novo registro basta ir em uma das páginas que listam os dados e clicar em adicionar um novo item.

### Campos

- Clientes -
  - Nome
  - Email
  - CPF
  - Endereço

- Produto - 
  - Nome 
  - Descrição
  - Quantidade
  - Custo
  - Preço
 
### Funcionalidades
 - cadastro de clientes 
 - atualização e exclusão de dados dos clientes

 - cadastro de produtos
 - atualização e exclusão dos dados dos produtos


# Armazenamento
 - Todos os dados são armazenados no local storage do browser

## Instalação

### Pré-requisitos

Antes de começar, você vai precisar ter instalado em sua máquina a seguintes ferramentas:
[Git](https://git-scm.com) e o [Live Server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer). 
Além disto é bom ter um editor para trabalhar com o código como [VSCode](https://code.visualstudio.com/)

```bash
# Clone este repositório
$ git clone <https://github.com/wendersoncirilo/desafio-2.git>

# Acesse a pasta do projeto no terminal/cmd
$ cd desafio-2

# Execute a aplicação usando o live server

```
